# Campaigns README

> **Purpose**
>
> This document defines how campaigns are organized within the
> repository and how campaign-specific knowledge should be interpreted.
>
> Each campaign represents an independent collection of approved
> information built upon the shared framework.

------------------------------------------------------------------------

# Campaign Philosophy

A campaign is the canonical home for everything unique to that story.

Campaigns inherit the global framework but define their own:

-   Characters
-   Locations
-   Items
-   Logos
-   Campaign-specific lore

Campaign information never changes the framework---it specializes it.

------------------------------------------------------------------------

# Campaign Structure

Each campaign should contain only information that belongs to that
campaign.

Example:

``` text
Campaign_Name/
├── CAMPAIGN.md
├── CHARACTERS.md
├── ITEMS.md
├── LOCATIONS.md
└── LOGOS.md
```

Additional documents may be added if approved and they serve a unique
purpose.

------------------------------------------------------------------------

# Ownership

Each campaign owns its approved content.

Characters, locations, items, and logos belong to the campaign in which
they were created unless explicitly documented otherwise.

Information should not be copied between campaigns. If something is
shared, it should be intentionally referenced rather than duplicated.

------------------------------------------------------------------------

# Relationship to the Framework

The Framework provides:

-   Artistic philosophy
-   Visual language
-   Global rules

The Campaign provides:

-   Identity
-   Setting
-   Approved details
-   Campaign-specific creative decisions

Campaign documents expand the framework without replacing it.

------------------------------------------------------------------------

# AI Responsibilities

When working within a campaign, ChatGPT should:

1.  Read the campaign documentation.
2.  Use campaign information as the canonical source for
    campaign-specific details.
3.  Combine campaign knowledge with the global framework.
4.  Never invent campaign facts.
5.  Ask clarifying questions if information is missing.

------------------------------------------------------------------------

# Campaign Continuity

Campaign documents preserve continuity over time.

Approved information should remain consistent across every future image,
prompt, and document.

Changes require explicit user approval.

Once approved, campaign information becomes part of the campaign's
permanent history.

------------------------------------------------------------------------

# Future Expansion

Each new campaign should:

-   Reuse the approved templates.
-   Follow the global framework.
-   Maintain its own independent identity.
-   Preserve its own continuity.

The repository is designed to support multiple campaigns while
maintaining a unified artistic standard across the entire collection.
