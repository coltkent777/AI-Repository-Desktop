# Workflow README

> **Purpose**
>
> This document defines how ChatGPT should use this repository before
> generating artwork or creating new repository content.
>
> It establishes the operating procedure for every session to ensure
> consistency, continuity, and adherence to the approved framework.

------------------------------------------------------------------------

# Workflow Philosophy

The repository is a structured knowledge base.

ChatGPT should gather approved information from the repository and
assemble it into a complete image-generation prompt. The user should not
need to restate information that already exists in approved documents.

If required information does not exist, ChatGPT should ask clarifying
questions before proceeding.

------------------------------------------------------------------------

# Standard Operating Procedure

For every new request:

1.  Read the repository `README.md`.
2.  Read `framework/README.md`.
3.  Read `workflow/README.md`.
4.  Read `campaigns/README.md`.
5.  Read the relevant campaign documents.
6.  Identify all referenced Character, Location, Item, and Logo
    documents.
7.  Resolve conflicts or ask clarifying questions if information is
    missing.
8.  Assemble the final prompt.
9.  Generate the requested artwork or content.

------------------------------------------------------------------------

# Prompt Assembly Process

The final image prompt should be assembled using the following order:

1.  Framework Rules
2.  Campaign Information
3.  Character Information
4.  Location Information (if applicable)
5.  Item Information (if applicable)
6.  Logo Information (if applicable)
7.  Current User Request

The current request should provide only scene-specific instructions such
as pose, composition, expression, camera angle, or environment
selection. Persistent information should come from the repository.

------------------------------------------------------------------------

# Prompt Generation Responsibility

ChatGPT is responsible for creating the final prompt submitted to the
image generator.

The user is not expected to manually combine repository documents.

The assembled prompt should faithfully represent the approved repository
information while incorporating only the new instructions provided in
the current request.

------------------------------------------------------------------------

# Repository Updates

When creating or modifying repository content:

-   Present drafts before finalizing.
-   Accept revisions.
-   Wait for explicit approval.
-   Once approved, treat the document as locked.
-   Never modify locked content without explicit user approval.

------------------------------------------------------------------------

# Conflict Resolution

If information conflicts:

1.  Stop.
2.  Identify the conflict.
3.  Ask the user for clarification.
4.  Do not choose an interpretation without approval.

------------------------------------------------------------------------

# Guiding Principles

-   Never assume missing information.
-   Ask questions instead of guessing.
-   Consistency takes priority over novelty.
-   Preserve approved artistic intent.
-   Build upon existing knowledge instead of recreating it.
-   Respect locked documents at all times.
