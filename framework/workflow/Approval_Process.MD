# Purpose

This document defines the approval process for modifying the Art Framework, campaigns, characters, and generated artwork.

The approval process exists to preserve consistency, prevent artistic drift, and ensure that established canon is never unintentionally altered.

---

# Approval Philosophy

Nothing within the framework should change without explicit user approval.

Conversation, brainstorming, and recommendations do not constitute approval.

Approval requires a clear affirmative decision from the user.

---

# Approval Levels

## Framework

Framework documentation is permanent.

Any modification to a framework document requires explicit user approval before the document is updated.

---

## Campaign

Campaign documentation is considered canon for that campaign.

Changes require explicit user approval.

---

## Character

Character documentation defines the canonical identity of that character.

Changes require explicit user approval.

---

## Artwork

Artwork generation may proceed using approved documentation.

If requested artwork conflicts with established canon, clarification should be requested before generation.

---

# Approved Actions

The following actions may be performed without additional approval:

- Reading framework documents.
- Referencing approved documentation.
- Constructing prompts from approved documentation.
- Making recommendations.
- Asking clarifying questions.
- Identifying potential inconsistencies.

These actions do not modify canon.

---

# Actions Requiring Approval

Explicit approval is required before:

- Editing framework documents.
- Editing campaign documentation.
- Editing character documentation.
- Replacing approved artwork direction.
- Establishing new permanent rules.
- Removing existing rules.
- Overriding established canon.

---

# Recommendation Process

Recommendations should remain separate from canon.

Recommendations should:

- Explain the proposed improvement.
- Wait for user approval.
- Be documented only after approval.

Recommendations are not automatically adopted.

---

# Canon Protection

Existing documentation should always be treated as the source of truth.

When uncertainty exists:

- Reference existing documentation.
- Ask for clarification.
- Do not assume.
- Do not overwrite existing decisions.

Preserving canon always takes priority over speed.

---

# Framework Integrity

Every permanent change should be intentional.

The framework should evolve through deliberate decisions rather than gradual drift.
