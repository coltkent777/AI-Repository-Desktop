# Purpose

This document defines the review process for generated artwork.

The review process exists to verify quality, preserve canon, identify potential improvements, and determine whether feedback applies only to the artwork or requires documentation updates.

---

# Review Philosophy

Every review should answer two questions:

1. Is the artwork successful?
2. Does the feedback affect canon?

Artwork quality and canon are reviewed independently.

---

# Artwork Review

Verify the following:

- Framework standards were followed.
- Campaign standards were followed.
- Character standards were followed.
- Composition is correct.
- Lighting is correct.
- Color language is correct.
- Visual language is correct.
- Character Manifestations are appropriate.
- Readability is maintained.
- No rendering defects are present.

---

# Canon Review

For every suggested revision, determine:

- Is this only an improvement to this artwork?
- Does this establish new permanent character information?
- Does this establish new campaign information?
- Does this establish a new framework rule?

If the answer is **no** to all permanent changes, no documentation should be updated.

---

# Documentation Impact

If feedback establishes new canon:

Determine which document should be updated.

Possible locations include:

- Framework document
- Campaign document
- Character document

Documentation should only be updated after explicit user approval.

---

# Canon Preservation

Do not update documentation simply because artwork changed.

Artwork revisions do not automatically establish canon.

Only documented decisions become permanent.

---

# Quality Checklist

Before approval, verify:

- Character identity is preserved.
- Character class is immediately recognizable.
- Signature color palette is respected.
- Character Manifestations reinforce identity.
- Visual hierarchy is clear.
- Readability is maintained.
- Background follows framework standards.
- Magical language is consistent.
- No unnecessary visual clutter exists.

---

# Approval Outcome

Every review should conclude with one of the following:

### Approved

Artwork satisfies framework standards.

No documentation changes required.

---

### Approved with Documentation Update

Artwork is approved.

Explicit user approval has established new canon.

Relevant documentation should be updated.

---

### Revision Required

Artwork requires revision before approval.

Canon remains unchanged.
