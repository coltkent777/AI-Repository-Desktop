# Purpose

This document defines the standard process for constructing prompts using the Art Framework.

The protocol exists to maximize consistency, preserve canon, reduce interpretation, and prevent artistic drift.

Every prompt generated using this framework should follow these standards.

---

# Prompt Hierarchy

Every prompt should be constructed using the following order.

1. Framework Rules
2. Campaign Rules
3. Character Rules
4. Artwork Request
5. Technical Output Requirements

Higher levels establish permanent standards.

Lower levels add context without replacing higher-level rules unless explicitly approved.

---

# Rule Priority

When conflicting instructions exist, priority is determined in the following order.

1. Framework
2. Campaign
3. Character
4. Current User Request

Existing canon should remain unchanged unless the user explicitly approves a modification.

---

# Modular References

Prompts should reference existing framework documentation rather than repeating it.

Whenever possible:

- Reference established standards.
- Reuse documented decisions.
- Avoid duplicate instructions.
- Avoid redefining existing concepts.

Each rule should exist in only one location within the framework.

---

# Required Information

The framework should never invent required information.

If essential information is missing:

- Stop prompt construction.
- Ask clarifying questions.
- Wait for user approval before proceeding.

Assumptions should never replace documented decisions.

---

# Canon Preservation

Framework canon is permanent until explicitly modified.

Campaign canon is permanent until explicitly modified.

Character canon is permanent until explicitly modified.

Prompt generation should preserve all established documentation unless directed otherwise by the user.

---

# Prompt Optimization

Prompts should be optimized for image generation models.

Priority should be given to:

- Accuracy
- Consistency
- Readability by the model
- Reduced ambiguity
- Efficient token usage

Human readability is desirable but secondary.

---

# Prompt Construction Principles

Every prompt should:

- Build upon existing documentation.
- Preserve established visual language.
- Reinforce campaign identity.
- Reinforce character identity.
- Follow framework standards.
- Avoid unnecessary repetition.

The objective is consistency rather than novelty.

---

# Framework Integrity

Framework documents remain the single source of truth.

Prompt generation should never create unofficial framework rules.

Any permanent rule changes require explicit user approval before documentation is updated.
